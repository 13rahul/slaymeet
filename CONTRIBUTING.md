# Contributing

Thanks for helping improve SlayMeet.

1. Read the [Code of Conduct](CODE_OF_CONDUCT.md)
2. Fork and create a branch
3. Keep changes focused; match existing PHP/JS style
4. Run `php -l` on changed PHP files
5. Do not commit `.env`, API keys, or customer data
6. Open a PR using the template (summary + test plan)

Accessibility-friendly fixes (keyboard, screen readers, contrast, captions) are especially welcome.

## Good first contributions

- Docs / Docker clarity
- UI polish and a11y
- Teena wake-word / language improvements
- Bug reports with reproduction steps
